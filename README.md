# Brandon Tuttle - GitHub Pages Site

This repository contains Brandon's first personal GitHub Pages site. It is a
plain HTML and CSS site with no build step, no analytics, and no third-party
widgets.

The site is intended to be a lightweight developer home base for open-source
collaboration, AI systems notes, robotics learning, LLM inference and
optimization interests, and a small amount of personal field-notes context.

## Files

- `index.html` - the single-page site.
- `404.html` - a friendly not-found page for GitHub Pages.
- `assets/css/styles.css` - all site styling.
- `assets/img/field-notes-workspace.jpg` - local hero image.
- `assets/img/favicon.png` - local favicon.
- `notes/README.md` - a placeholder for future notes.

## Editing Locally

Because this version has no build step, edit the files directly and open
`index.html` in a browser.

For a quick local server:

```bash
python3 -m http.server 8000
```

Then visit `http://localhost:8000`.

## Publishing

For a GitHub user site, the repository should be named:

```text
tuttlebr.github.io
```

GitHub Pages can publish the site from the repository root. Once enabled, the
default public URL is:

```text
https://tuttlebr.github.io
```

The expected public source repository URL is
`https://github.com/tuttlebr/tuttlebr.github.io` after the repository has been
created or made public.

## Content Notes

The contact section currently publishes only public GitHub links. Add email,
LinkedIn, Bluesky, Mastodon, or other contact paths only if they are safe to be
indexed publicly.

Project cards are intentionally honest placeholders where work is planned or
still forming.
